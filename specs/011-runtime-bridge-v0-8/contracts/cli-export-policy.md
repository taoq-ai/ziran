# CLI Contract: `ziran export-policy`

## Synopsis

```
ziran export-policy --result <path> --format <format> [--out <dir>] [--severity-floor <level>] [--verbose]
```

## Options

| Option | Type | Required | Default | Description |
|--------|------|----------|---------|-------------|
| `--result` | `Path` | Yes | — | Campaign result JSON file |
| `--format` | `Choice` | Yes | — | One of: `rego`, `cedar`, `nemo`, `invariant` |
| `--out` | `Path` | No | `./policies/` | Output directory for generated files |
| `--severity-floor` | `Choice` | No | `medium` | Minimum severity to export: `critical`, `high`, `medium`, `low` |
| `--verbose` | `Flag` | No | `False` | Show per-finding details during export |

## Behavior

1. Load campaign result from `--result`.
2. Filter `dangerous_tool_chains` to those at or above `--severity-floor`.
3. For each qualifying finding, invoke the format-specific renderer.
4. Write each policy to `--out/<finding_id>.<ext>` (extension per format: `.rego`, `.cedar`, `.co`, `.invariant`).
5. If a finding cannot be expressed in the target format, skip it and log the reason.
6. Print summary: `Exported N policies. Skipped M findings (N below severity floor, K unsupported for format).`

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success (at least one policy exported) |
| 0 | Success (no qualifying findings — informational message) |
| 1 | Error (file not found, invalid JSON, invalid format) |

## Output Files

Each file begins with a comment header:

```
# Generated by Ziran v0.8
# Finding: <finding_id>
# Chain: <tool_a> → <tool_b> [→ ...]
# Severity: <severity>
```
